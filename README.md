# TPS Loot Prototype

아이템 **탐색 / 습득 / 장착 / 이동 / 교체** 인터랙션을 PC와 콘솔(Xbox 패드)에서 심리스하게 검증하기 위한 프로토타입입니다. 단일 HTML 파일 + Three.js로 구성, 패드를 꽂거나 마우스를 움직이면 화면 힌트가 자동으로 전환됩니다.

---

## 인터랙션 가이드

### 1. 캐릭터 / 월드

| 액션 | PC | Pad |
|---|---|---|
| 이동 | `WASD` | 좌측 스틱 |
| 시점 | 마우스 | 우측 스틱 |
| 달리기 / 점프 / 사격 | `Shift` / `Space` / 좌클릭 | `RB` / `A` / `RT` |
| 1·3인칭 전환 | `C` | — |

### 2. 바닥 아이템 & 상자

거리에 따라 3단계: **3m 흰 원(인지) → 1.5m + 카메라 향함 → prompt 표시**.

- 바닥 아이템: `E` (PC) / `X` (Pad) **탭** → 즉시 픽업
- 상자/시체: 같은 키 **2초 꾹** → ring 차오르며 탐색 → 인벤이 열리고 좌측 "주변" 영역에 내용물 표시
- prompt 의 키 모양: 흰 원 + 검은 글자 = 탭 / 흰 원 + 보더 + ▼ = 꾹

### 3. Quick Loot 휠 (투척/회복 빠른 사용)

- PC: `Q` 꾹 → 마우스로 슬라이스 → `Q` 떼면 사용
- Pad: `LB` 꾹 → 우측 스틱으로 슬라이스 → `LB` 떼면 사용

### 4. 인벤토리 — `Tab` (PC) / `Y` (Pad)

화면 3컬럼: **주변** (바닥+컨테이너) / **배낭** (+안전포켓 2슬롯) / **장비** (무기1·2 + 부착물 + 헬멧·조끼·가방).

#### 기본 조작

| 액션 | PC | Pad |
|---|---|---|
| 슬롯 포커스 | 마우스 호버 | D-pad / 스틱 |
| 영역 순환 | 클릭 | `LB` / `RB` |
| 장착·회수·픽업 | 더블 클릭 | `A` |
| 슬롯 이동·교체·버리기 | 드래그-드롭 | (아래 picker 참조) |
| 툴팁 정보 ↔ 액션 토글 | 우클릭 | `Y` |
| 액션 버튼 실행 | 클릭 | `A` |
| 인벤 닫기 | `Esc` | `B` / `Start` |

> 옵션 "**인벤 열려도 캐릭터 이동**" ON (기본): 인벤 중에도 캐릭터 이동 가능. OFF 면 좌측 스틱/WASD 가 슬롯 nav.

### 5. 아이템 툴팁 — 정보 / 액션 두 모드

마우스 호버 또는 패드 포커스 시 슬롯 옆 카드.
**우클릭 / Y** 로 정보 ↔ 액션 토글. 출발 위치에 따라 액션 종류가 다름.

| 출발 | 액션 |
|---|---|
| 주변 | 가져가기, 나누기 (qty>1), 아이템 핑, 안전 포켓으로 이동 |
| 배낭 | 아이템 핑, 장착하기, 나누기, 안전 포켓으로 이동, 버리기 |
| 장비 (무기) | 아이템 핑, 회수하기, 안전 포켓으로 이동, 버리기, 무기 교체 (1↔2번) |
| 장착된 부착물 | 아이템 핑, 회수하기, 안전 포켓으로 이동, 버리기 |
| 안전 포켓 | 배낭으로 이동, 버리기 |

### 6. 부착물 / 무기 슬롯 선택 (Pad Picker 모드)

패드로 부착물 또는 무기에 `A` (또는 액션 메뉴의 "장착하기"/"회수하기") 누르면 자동 장착 대신 **picker 모드** 진입:

- 이동 가능한 슬롯에 **흰 펄스(반투명 깜빡임)** 표시, 자동으로 가장 위쪽에 포커스
- D-pad / 스틱으로 이동 → `A` 확정 / `B` 취소

**스킵 케이스 (picker 안 거치고 즉시 처리)**
- 무기 + 무기 슬롯이 하나라도 비어있으면 → 빈 자리에 바로 장착
- 방어구(헬멧·조끼·가방) → 빈 슬롯이면 장착, 점유 슬롯이면 즉시 교체

### 7. 부착물 호환성

같은 부위라도 무기 클래스가 다르면 장착 불가:

- **AR**: 소염기(AR), 보정기(AR) 등 → 라이플/스나이퍼에만
- **SMG**: 소염기(SMG) → SMG에만
- 사이트 / 그립 / 탄창은 무기 종류 무관

**자동 장착은 빈 슬롯에만**. 이미 부착물이 있는 슬롯에 같은 종류 부착물을 픽업해도 자동 교체 안 됨 — 직접 드래그 또는 picker 로 교체.

### 8. 안전 포켓 (배낭 하단 2슬롯)

무게/슬롯 제한 영향 없는 보조 저장. 어떤 아이템이든 드래그·드롭 또는 액션 메뉴로 이동.
무기를 넣으면 부착물이 자동 분리되어 배낭으로 들어감.

### 9. 호환·업그레이드 시각 표시

- 슬롯 우상단 **초록 점**: 현재 장착 무기와 호환되는 탄약/부착물
- 픽업 prompt 하단 **가로 무기 실루엣**: 어떤 무기에 맞는지 표시
- prompt 우측 **↑ 화살표**: 더 좋은 등급 (자동 교체 후보)

## 실행

```bash
npm install
npm run dev
```

브라우저에서 `http://localhost:5173/` — Vite HMR 로 파일 수정 시 자동 리로드.

## 공유

| 방법 | 명령 / 절차 |
|---|---|
| **LAN 시연** (가장 빠름) | `npm run dev -- --host` → Network 주소를 같은 망의 동료에게 |
| **정적 호스팅 빌드** | `npm run build` → `dist/` 를 Netlify/Vercel/GitHub Pages/사내 호스트에 업로드 |
| **Netlify CLI 배포** | `netlify deploy --prod --dir=dist` (figma-assets 포함, public/ 가 자동으로 dist/ 에 복사됨) |
| **빌드 미리보기** | `npm run preview -- --host` |
| **HTML 파일 직접 전달** | `시뮬레이터_루팅_시스템_Prototype.html` 단일 파일만 전달 — 브라우저에서 바로 열림 (CDN 접속 가능 환경 한정) |

## 구조

```
시뮬레이터_루팅_시스템_Prototype.html   # 본체 (CSS/JS/HTML 모두 인라인, Three.js r128 CDN)
index.html                              # localhost:5173 → 본체로 리다이렉트
public/figma-assets/                    # 아이콘·무기 렌더 (cat-icons / pubg-weapons / tooltip-icons / pubg-icons) — vite 가 빌드 시 dist/ 루트로 자동 복사
vite.config.js                          # build 시 두 HTML 모두 dist 에 포함
package.json                            # vite devDependency 하나
```
