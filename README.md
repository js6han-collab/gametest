# TPS Loot Prototype

PUBG 스타일 3인칭 슈터 루팅/인터랙션 프로토타입.

진실의 원천(source of truth)은 단일 파일 **[시뮬레이터_루팅_시스템_Prototype.html](시뮬레이터_루팅_시스템_Prototype.html)** 입니다. CSS/JS/HTML 모두 인라인이며 Three.js r128을 CDN에서 로드합니다. 이 저장소는 그 파일을 편집하고 공유하기 편하도록 Vite를 dev 서버로 얹어 둔 것일 뿐입니다.

## 실행

```bash
npm install
npm run dev
```

브라우저에서 [http://localhost:5173/](http://localhost:5173/) 열기 → 자동으로 프로토타입 파일로 리다이렉트됩니다.

프로토타입 HTML을 편집하면 Vite가 브라우저를 자동 리로드합니다.

## 동료에게 공유

### 옵션 1 — LAN 시연 (가장 빠름)

```bash
npm run dev -- --host
```

표시되는 Network 주소를 같은 망의 동료에게 전달.

### 옵션 2 — 정적 호스팅용 빌드

```bash
npm run build
```

`dist/` 에 `index.html` 과 `시뮬레이터_루팅_시스템_Prototype.html` 이 함께 생성됩니다. Netlify/Vercel/GitHub Pages/사내 정적 호스트 등에 그대로 업로드.

```bash
npm run preview -- --host
```

로컬에서 빌드 결과물을 LAN으로 미리보기.

### 옵션 3 — 파일만 직접 전달

프로토타입 HTML은 완전 독립형(self-contained)이므로 `시뮬레이터_루팅_시스템_Prototype.html` 한 파일만 동료에게 보내도 브라우저에서 바로 열립니다. (CDN 접속 가능한 환경 한정.)

## 인터랙션 가이드

키보드/마우스(PC) 와 Xbox 게임패드 둘 다 지원합니다. 패드 입력이 감지되면 화면 힌트(키 아이콘, 토글 표시 등)가 자동으로 패드 모드로 전환됩니다. 마우스를 움직이면 다시 PC 모드로 돌아갑니다.

### 1. 월드 — 캐릭터 조작

| 액션 | PC | Pad |
|---|---|---|
| 이동 | `W A S D` | 좌측 스틱 |
| 시점 회전 | 마우스 | 우측 스틱 |
| 달리기 | `Shift` | `RB` |
| 점프 | `Space` | `A` |
| 앉기 | `C` | — |
| 1·3인칭 전환 | `C` 토글 (앉기와 무관) | — |
| 사격 | 좌클릭 | `RT` |

### 2. 월드 — 바닥 아이템 & 상자 인터랙션

거리에 따라 **3 단계**로 표시가 바뀝니다.

1. **3m 안 — 인지 단계**: 아이템·상자 위에 흰 테두리 원이 뜸. "여기 뭐가 있다" 만 알려줌.
2. **1.5m 안 + 카메라가 향함 — prompt 단계**: 원이 사라지고 버튼 가이드 표시.
   - 바닥 아이템: `E` (PC) / `X` (Pad) 한 번 → 즉시 픽업
   - 상자/시체: **2초 꾹** 누르면 진행 ring 이 차오르며 탐색 → 인벤이 열리고 좌측 "주변" 영역에 내용물이 들어옴
3. prompt 의 키 모양으로 동작 구분
   - 흰 원 + 검은 글자 (보더 없음) = 한 번 탭
   - 흰 원 + 보더 + 상단 ▼ 표시 = 꾹 누르기

> 상자 탐색 중 키를 떼면 진행 취소. 탐색 후 1.5m 밖으로 벗어나면 "주변" 의 컨테이너 sub-section 자동 사라짐.

### 3. Quick Loot — 라디얼 휠 (투척물/회복 빠른 사용)

- **PC**: `Q` 꾹 → 휠 열림 → 마우스로 슬라이스 선택 → `Q` 떼면 사용
- **Pad**: `LB` 꾹 → 우측 스틱으로 슬라이스 선택 → `LB` 떼면 사용

### 4. 인벤토리 — 화면 구성

`Tab` (PC) / `Y` (Pad) 로 열기·닫기. `Esc` (PC) / `B` (Pad) 로도 닫힘.

화면은 **3개 컬럼**으로 나뉩니다.

| 컬럼 | 내용 |
|---|---|
| **좌측 — 주변** | 컨테이너 내용물 + 바닥 아이템 sub-section. 둘 다 비어도 "주변" 헤더는 항상 표시되고 우측에 세로 구분선이 그어짐. |
| **중앙 — 배낭** | 배낭 본체(N x 4 그리드) + 하단의 **안전 포켓** 2슬롯. |
| **우측 — 장비** | 1번/2번 무기 카드(부착물 슬롯 포함) + 헬멧/조끼/가방 + 소모품 카운터. |

### 5. 슬롯 기본 조작

#### PC (마우스 + 키보드)

| 액션 | 방법 |
|---|---|
| 슬롯 포커스 | 마우스 호버 |
| 빠른 이동 (장착/회수/픽업) | 더블 클릭 |
| 슬롯 간 이동 / 안전포켓 / 바닥 버리기 | 클릭 드래그 → 목적지에 드롭 |
| 다중 선택 | `Ctrl` + 좌클릭 |
| **아이템 툴팁 (정보 ↔ 액션) 토글** | 슬롯 우클릭 |
| 주변 아이템 한꺼번에 가져오기 | `E` 꾹 |

#### Xbox 패드

| 액션 | 버튼 |
|---|---|
| 슬롯 포커스 이동 | D-pad / 좌측 스틱 (또는 우측 스틱 — 옵션 따라) |
| 영역 순환 (주변 ↔ 배낭 ↔ 장비) | `LB` / `RB` |
| **장착·회수·픽업** | `A` |
| **아이템 툴팁 (정보 ↔ 액션) 토글** | `Y` |
| 액션 버튼 실행 (툴팁 action 상태 또는 picker 모드) | `A` |
| 픽업 전체 / 슬롯 비우기 | `X` 꾹 (주변/컨테이너 포커스 = 전체 가져오기 / 배낭·장비 포커스 = 바닥 버리기) |
| 인벤 닫기 | `B` 또는 `Start` |

> "인벤 열려도 캐릭터 이동" 옵션(우측 사이드패널)이 켜져 있으면 인벤 중에도 좌측 스틱 = 캐릭터 이동, 우측 스틱 = 슬롯 nav. 끄면 좌측 스틱이 슬롯 nav. 기본 ON.

### 6. 아이템 툴팁 — 정보 / 액션 두 모드

마우스 호버 또는 패드 포커스 시 슬롯 옆에 작은 카드가 뜹니다.

- **정보 모드 (기본)**: 등급(고급/희귀/엘리트/전설) + 카테고리, 설명, 최대 묶음, 태그(SMG·AR·5.56mm 등), 무게/가치
- **액션 모드 (토글 시)**: 해당 슬롯에서 가능한 액션 버튼 목록 — 마우스 호버 또는 패드 포커스된 버튼만 검은 배경 + 흰 글자

토글은 **우클릭** (PC) 또는 **Y** (Pad). 풋터 작은 핀(좌하단)이 현재 모드에서 토글하면 어떻게 되는지 알려줍니다 — `액션` 또는 `뒤로`.

#### 액션 종류 — 출발 위치별

| 출발 | 표시되는 액션 |
|---|---|
| **주변(바닥/컨테이너)** | 가져가기, 나누기 (qty>1), 아이템 핑, 안전 포켓으로 이동 (빈 칸 있을 때) |
| **배낭** | 아이템 핑, 장착하기 (장착 가능 시), 나누기, 안전 포켓으로 이동, 버리기 |
| **장비 (무기·방어구)** | 아이템 핑, 회수하기, 안전 포켓으로 이동, 버리기, 무기 교체 (1번 ↔ 2번 무기) |
| **부착물 (총기에 장착된)** | 아이템 핑, 회수하기, 안전 포켓으로 이동, 버리기 |
| **안전 포켓** | 배낭으로 이동, 버리기 |

### 7. 패드 — 슬롯 선택 (Picker) 모드

부착물/무기에 `A` 또는 액션 메뉴의 "장착하기"/"회수하기" 를 누르면 자동 장착되는게 아니라 **picker 모드** 로 들어갑니다.

- **이동 가능한 슬롯에만 흰 펄스(반투명 깜빡임) 표시**, 나머지는 그대로 (포커스 차단)
- 자동으로 가장 위쪽의 호환 슬롯에 포커스
- D-pad / 스틱으로 이동 → `A` 확정 / `B` 취소

#### Picker가 보여주는 슬롯

| 상황 | 하이라이트되는 슬롯 |
|---|---|
| 배낭/주변/안전포켓의 **부착물** | 부착 가능한 무기 카드의 attach 슬롯들 (호환 안 되면 표시 안 됨) |
| **장착된 부착물** (무기에 끼워진) | 다른 무기의 호환 attach 슬롯 + 배낭 첫 빈 슬롯 1개 (배낭 회수용) |
| 배낭/주변의 **무기** (양쪽 다 비어있지 않을 때) | 1번 + 2번 무기 슬롯 (교체용) |
| **장착된 무기** | 반대편 무기 슬롯 + 배낭 첫 빈 슬롯 1개 |

#### Picker 스킵 케이스

- **무기 + 비어있는 무기 슬롯이 있을 때** — picker 안 뜨고 바로 빈 슬롯에 장착
- **방어구 (헬멧/조끼/가방)** — 빈 슬롯이면 바로 장착, 점유 슬롯이면 즉시 교체 (기존 장비는 배낭으로 회수)

### 8. 부착물 호환성

`소염기(AR)` 와 `소염기(SMG)` 처럼 동일 부위라도 무기 클래스가 다르면 **장착 불가** — 픽업해도 자동 장착 안 됨, 드래그해도 거부됨(빨간 invalid). 호환 클래스:

- **AR-class** (rifle/sniper): `소염기 AR`, `보정기 AR` 등
- **SMG-class**: `소염기 SMG` 등
- 사이트/그립/탄창 — 무기 종류 무관 (universal)

> **자동 장착은 빈 슬롯에만** — 이미 부착물이 있다면 픽업해도 자동 교체 안 됨. 교체하려면 명시적으로 드래그 또는 picker 모드로 진행.

### 9. 드래그/드롭 동작

마우스로 슬롯을 잡고 끌면 가능한 드롭 타겟에 흰 테두리가 뜹니다.

| 출발 → 목적지 | 결과 |
|---|---|
| 주변 → 배낭 빈 칸 / 컬럼 / 안전포켓 / 무기·방어구 슬롯 | 즉시 픽업 (자동 분배 또는 명시 슬롯 장착) |
| 배낭 → 무기/방어구 슬롯 | 장착 (점유 시 swap — 기존 아이템은 배낭으로) |
| 배낭/장비 → 좌측 컬럼 | 바닥에 버리기 |
| 배낭/장비 → 중앙 컬럼 빈 영역 | 배낭으로 회수 |
| 부착물(1번 무기) → 2번 무기 sub-slot (호환) | 1:1 swap |
| 부착물(1번 무기) → 2번 무기 sub-slot (반대로 비호환) | 1번 슬롯은 비고, displaced 는 배낭 |
| 안전포켓 → 어디든 / 어디든 → 안전포켓 | 모두 가능. 무기를 안전포켓 넣으면 부착물은 자동 분리되어 배낭으로 |

### 10. 안전 포켓

배낭 본체와 별개로 2슬롯의 보조 저장 공간. 무게/슬롯 제한 영향 없이 임의 아이템을 넣어둘 수 있습니다.

- 드래그 또는 액션 메뉴의 "안전 포켓으로 이동" 으로 진입
- 안전포켓 아이템 액션은 **배낭으로 이동** / **버리기** 두 가지만 노출
- 무기를 안전포켓에 넣으면 그 무기의 부착물은 자동 분리되어 배낭으로 이동

### 11. 호환성 / 업그레이드 시각 표시

- **인벤 슬롯 우상단 초록 점**: 이 아이템이 장착 무기와 호환 (탄약 calibre 일치 또는 부착물 weaponClass 일치)
- **바닥 픽업 prompt 하단 가로 무기 실루엣**: 포커싱한 탄약·부착물이 호환되는 무기 표시
- **prompt 우측 ↑ 화살표**: 더 좋은 등급 무기/방어구가 있을 때

### 12. 패드 — 슬롯 간 이동 디테일

D-pad / 스틱으로 슬롯 이동 시 spatial nav 가 기본이지만 몇 가지 명시 룰이 우선:

- 배낭 가장 오른쪽 끝 아이템 → **오른쪽** → 1번 무기 (있으면) → 2번 → 방어구
- 1번/2번 무기 → **오른쪽** → 그 무기의 가장 오른쪽 채워진 부착물
- 1번/2번 무기 → **아래** → 그 무기의 가장 왼쪽 채워진 부착물 (없으면 다음 무기 → 방어구)
- 무기 부착물 → **위** → 항상 그 무기 본체
- 1번 무기 부착물 → **아래** → 2번 무기 (없으면 방어구)
- 2번 무기 부착물 → **아래** → 방어구
- 무기 또는 가장 왼쪽 부착물 → **왼쪽** → 가까운 배낭 슬롯 (배낭 비면 주변)

> 빈 attach 슬롯과 빈 배낭 칸은 패드 포커스에서 자동 스킵됩니다 (드래그/드롭 타겟으로는 여전히 사용 가능). 장비 영역에 아이템이 하나도 없으면 `LB/RB` 사이클이 그 패널을 건너뜁니다.

### 13. 보조 키 (PC)

- `O` — 우측 사이드패널(프로토타입 컨트롤 옵션) 토글
- `Tab` — 인벤토리 열기/닫기
- `Esc` — 인벤토리 닫기

### 14. 폴리시 디테일

- 카메라 walk bob + idle breathing (1인칭 한정)
- 사격 시 FOV 펀치 + crosshair spread 동적 확장
- 무기 카드 좌하단 = 등급 색상 강조, 우하단 = 탄약 아이콘 + 잔탄/최대
- 부착물 빈 슬롯 = 흰색 반투명 placeholder (장착 가능한 자리 힌트)
- 슬롯 포커스/호버 시 무지개 그라데이션 ring (rarity 와 별개의 시각 cue)
- 툴팁이 떠있는 슬롯은 위 ring 이 계속 표시되어 어떤 아이템 정보인지 명확
- 컨테이너 탐색 직후 인벤 자동 오픈 시 포커스가 컨테이너 첫 슬롯으로 이동
- 패드로 주변에서 `X` 픽업 시 포커스가 "가장 위" 가 아니라 **방금 가져간 자리 근처** 의 잔여 아이템으로 유지
